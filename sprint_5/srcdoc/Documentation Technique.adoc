== Introduction

OPTI es un logiciel permettant de gérer les projets tutorés. +
Cette documentation vous fournira toutes les informations nécessaires pour bien comprendre le fonctionnement de l'application. Vous pouvez toutefois consulter la *documentation utilisateur* concernant d'autres informations sur le sujet. 

include::backLog.txt[]

=== Sprint 3

La sujet est abordé en réalisant:

	- L'intégration des fonctions d'affichage et gestion des rôles, voeux et associations. 
	
	- Le test des fonctions d'affichage et de gestion des données.

== Fonctionnalités 

- *Make.bat*:

* Pour pouvoir générer la documentation et lancer les tests automatiques il suffit d'éxécuter le fichier make.bat qui se trouve dans la racine du sprint. +
La documentation sera générée dans le répertoire nommé *"doc"*. +


image::make.png[link="../make.bat", ,width="50%"] 

- *Gérer les différents acteurs des PTUTs:*
* Ajouter un acteur

* Modifier un acteur
* Supprimer un acteur
* Cloner un projet
* Assigner un rôle à un intervenant

- *Assigner et gérer les différents projets:*
* Créer un projet
* Assigner un(des) étudiant(s) à un projet
* Supprimer un projet
* Assigner un projet à un groupe



- *Gérer les informations destinées aux intervenants:*
* Afficher les informations liées à un projet, étudiant ou sujet
* Sauvegarder ou Charger des données
* Noter et évaluer les projets*



- *Onglet "A propos"*:

* Onglet de l'application qui contient les informations concernant les auteurs. +

- *Quitter*:

* Permet de quitter l'application. +

=== Choix techniques

Utilisation d'une classe abstraite *OptiElement* dont héritent les sous-classes *Intervenant*, *Etudiant*, *Projet* et *Sujets*. Cette strucuture permet de gérer plus facilement les différents éléments en les traitant comme une même donnée. +

----
include::..\OPTI\src\fr\iut_blagnac\data\model\OptiElement.java[OptiElement.java] 
----

De plus des contrôles de saisie de texte sont implémentés dans le code source des différentes classes. Comme on peut bien voir avec la méthode isValid() dans l'exemple suivant de la classe *Intervenants*:
----
include::..\OPTI\src\fr\iut_blagnac\data\model\Intervenant.java[Intervenant.java] 
----
La mise en page est gérée par le package *.gui*, dont plusieurs classes sont décrites pour mettre en place une intérface logique et érgonomique. 

Enfin, les données seront traités dans le .csv, pour cela on utilise la classe CSVManager: 
----
include::..\OPTI\src\fr\iut_blagnac\io\CSVManager.java[CSVManager.java] 
----
